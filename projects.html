Crux Compiler






Source has been lost 😕

I wrote an LLVM-based compiler for a simple language called Crux. The compiler is written in Java and uses real world tools like ANTLR4. The project covers all aspects of a compiler: grammar, the AST, static type checking, creating the IR, and generating x86_64 assembly code. The language itself supports function definition, recursion, control flow structures, and strong typing with static scoping with variable shadowing.

This was the most difficult project I have ever worked on at that point in time, but I loved every bit of it. It challenged all of the concepts of programming that I knew as well as introducing new concepts, like Reflection and using an IDE debugger. I kinda shot myself in the foot because this was a pair project that I chose to do by myself, but I enjoyed the challenge that came of it, so I didn't mind.

Parallel Rasterizer





Cannot release source

Created a Rasterizer in C++ that can trace primitive shapes in a given scene and outputs a PGM image. The program uses a custom file type to create the scene, where you can define the viewport size and location, light point, and location of shapes. I created this project using OOP principles with some new (at the time) features of C++17. The programing runs with an arbitrary amount of threads without encountering any data races or deadlocks and is about 70% faster on average than the single threaded version.

This was a fun project that I made over the course of a couple days. Ironically, the hardest parts of the project was reading the physics equations correctly. This was the first time I wrote a multi-threaded program, and it was very intimidating, but working on the project, as well as some other ones after, really demystified concurrency and helped turn it into a valuable skill.

During my 3rd year at UCI I took a class where we went over many different styles and features of different programming languages and patterns.   • 2) In each example, I had to use a lot of neat tricks to get around C++ static typing (including the first time I ever used constexpr). I think it is a good sign of my ingenuity as programmer and my ability to solve unusual problems.
